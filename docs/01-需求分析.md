# 01 - 需求分析

## 一、项目背景

### 1.1 我们要做什么

打造一个专属的"矿山投资 AI 助手"，部署在电脑桌面软件上，随时可用。它能：

- **读懂专业文档**：上传地质报告、可研报告等，基于文档内容回答问题
- **记住历史对话**：不会每次都从头开始，记得之前讨论过的项目
- **拥有专业知识**：比通用 AI 更懂矿业投资
- **能做计算**：IRR、NPV、储量估算等，不只是聊天

### 1.2 为什么要做

直接用 DeepSeek 等通用大模型存在三个核心问题：

| 问题 | 表现 | 影响 |
|------|------|------|
| **不懂行** | 没有矿业专业知识库，容易编造数据 | 回答不专业，误导决策 |
| **记不住** | 每次新对话从零开始 | 讨论过的项目、结论全部丢失 |
| **无法调用工具** | 不能运行代码计算 IRR | 无法完成实际分析工作 |

### 1.3 为什么从百炼平台迁移

**核心原因：百炼不是真正的 Agent，只是加了 RAG 的 Chatbot**

```
百炼（现状）：
用户问题 → RAG检索 → LLM回答 → 结束
           （一次性，不会反思、不会主动调用工具）

真正的 Agent（目标）：
用户问题 → 规划任务 → [循环开始]
              ↓
         需要知识？→ 调用 RAG
              ↓
         需要最新数据？→ 联网搜索
              ↓
         需要计算？→ 执行代码
              ↓
         信息够了吗？→ 不够，继续搜索
              ↓
         反思结果 → [循环继续直到满意]
              ↓
         输出最终答案
```

### 1.4 百炼的具体局限

| 场景 | 百炼能做的 | 真正 Agent 能做的 |
|------|-----------|------------------|
| "帮我分析这个铜矿" | 检索一次知识库，回答 | 检索 → 发现品位数据缺失 → 主动问你要 → 继续分析 |
| "铜价最新走势" | 调一次搜索 API | 搜索 → 发现信息矛盾 → 再搜几个来源 → 交叉验证 |
| "计算 IRR 并和其他项目对比" | 可能做不到多步骤 | 计算 IRR → 查知识库其他项目 → 生成对比表 → 给建议 |
| "这个项目值得投吗" | 基于检索的单次回答 | 按 7 大框架逐步评估 → 每步都可能需要多轮检索/搜索 |

## 三、使用场景

### 3.1 简单场景（单一矿山）

#### 场景 1：文档快速分析

```
用户：上传 80 页贵州磷矿地质勘查报告

提问：这个矿的资源量是多少？品位达标吗？有没有开采风险？

期望：AI 几秒内提取关键数据并给出初步判断
```

#### 场景 2：专业计算

```
用户：输入某铜矿的储量、品位、开采成本等参数

提问：帮我计算 NPV、IRR、投资回收期，并做敏感性分析（铜价波动±20%）

期望：AI 执行计算，输出结果和分析
```

#### 场景 3：综合评估

```
用户：上传某锰矿项目的地质报告、可研报告、环评报告

提问：按多个维度做完整评估，输出评分和投资建议

期望：AI 按 7 大框架（地质、水文、开发条件、采矿、选矿、经济性、合规风险）系统评估
```

### 3.2 复杂场景

#### 场景 4：知识库检索

```
案例 1：知识库存了选矿工艺手册
提问：处理含铜 0.5% 的斑岩型铜矿，浮选工艺的回收率一般能到多少？药剂方案怎么选？
期望：AI 从知识库检索相关章节给出专业回答

案例 2：知识库存了 20 个矿山项目资料
提问：我们看过的项目里，地质条件和品位信息是什么？
期望：AI 跨多份项目文档检索汇总

案例 3：横向对比
提问：把多个矿山放在一起比，从储量、品位、交通条件、开采成本等维度做表格对比
期望：AI 生成对比表格，和用户讨论
```

#### 场景 5：联网搜索

```
提问：2026 年锂矿最新价格走势如何？
提问：贵州省最新的矿业权出让政策有什么变化？
提问：帮我搜集国内其他铝土矿项目的资料

期望：AI 通过联网搜索获取最新公开信息，补充知识库中没有的实时数据
```

#### 场景 6：记忆回溯

```
背景：上个月跟 AI 讨论过一个甘肃的镍矿项目

提问：之前那个甘肃镍矿项目，我们的结论是什么？还有哪些待办事项？

期望：AI 能检索到历史摘要并回答
```

## 四、智能体能力要求

### 4.1 四大核心能力

| 组成部分 | 作用 | 对应产品功能 |
|---------|------|-------------|
| **大脑** | 理解问题、分析资料、给出判断 | AI 大模型（Claude/DeepSeek） |
| **规划** | 拆解步骤，先查资料再分析再给结论 | 提示词中角色定义、评估框架 |
| **工具** | 能动手干活—读文件、算数、搜网页 | 代码执行、联网搜索、RAG |
| **记忆** | 记住历史对话、上传的资料 | 知识库 + 聊天记录归档 |

### 4.2 专业评估框架（7 大维度）

#### A. 地质资源评估
- 矿体形态与规模（长度、宽度、厚度、延深、产状）
- 矿石品位（平均品位、边界品位、工业品位对比）
- 资源量/储量分级（333推断→332控制→331探明）
- 矿石类型与矿物组合、矿化连续性与变化系数

#### B. 水文地质与工程地质
- 含水层分布与地下水补给、矿坑涌水量预测
- 岩体稳定性（RQD、RMR分级）、地应力条件
- 区域特殊性：贵州喀斯特地下水复杂性、甘肃干旱区水源问题

#### C. 开发条件评估
- 交通（距公路/铁路/港口距离，运输成本）
- 电力（距电网距离、供电容量、电价）
- 水源（工业用水需求 vs 可用水量）
- 地形地貌、气候条件、劳动力与社区关系

#### D. 采矿工程评估
- 开采方式选择（露天/地下/联合，剥采比计算）
- 采矿方法与采场参数、损失率与贫化率
- 年产能设计与服务年限、设备选型与投资估算

#### E. 选矿工艺评估
- 可选性试验结果、工艺流程推荐（浮选/磁选/重选/氰化/堆浸）
- 回收率与精矿品位、尾矿处理方案（干排/湿排/膏体充填）
- 药剂消耗与成本

#### F. 经济性评估
- CAPEX（采矿+选矿+基建）、OPEX（采矿+选矿+管理+运输）
- 估值方法按项目阶段选择：
  - 早期探矿权 → 资源量法（$/吨，对标可比交易）
  - 详查阶段 → 可比交易法 + DCF 初步测算
  - 可研阶段 → DCF 为主（NPV、IRR、投资回收期）
  - 在产矿山 → 收益法（中国矿业权评估准则）
- 敏感性分析（价格±20%、品位±10%、成本±15%）
- 盈亏平衡分析

#### G. 合规与风险评估
- 采矿权/探矿权状态、有效期、权属清晰度
- 环评状态、安全生产许可
- 地质灾害风险、政策风险（资源税、环保趋势、碳中和）
- ESG（尾矿库安全、生态修复、社区影响）

```
┌─────────────────────────────────────────────────────────────┐
│                    云端服务                                  │
│  ┌──────────────────────────────────────────────────────┐  │
│  │           Deep Agents 智能体服务                      │  │
│  │  ├── LLM API（Claude / DeepSeek）                    │  │
│  │  ├── 百炼 RAG API（知识库检索）                       │  │
│  │  └── Tavily API（联网搜索）                          │  │
│  └──────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                              ↑↓ API
┌─────────────────────────────────────────────────────────────┐
│              Arcstone 桌面软件 (Electron)                    │
│  ├── 嵌入式 Python 环境（用户无需安装）                      │
│  ├── 本地代码执行（IRR/NPV 计算）                           │
│  ├── 本地记忆存储（SQLite）                                 │
│  └── 对话界面 + 知识库管理                                  │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 桌面软件功能规划

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 对话交互 | 与智能体自然对话 | P0 |
| 文件上传 | 在对话中上传文档供分析 | P0 |
| 知识库管理 | 批量上传文件到云端知识库 | P1 |
| 聊天记录归档 | 将重要对话归档到长期记忆 | P1 |
| 模型切换 | 在界面上切换不同 LLM | P2 |
| 项目档案查看 | 查看/编辑项目记忆文件 | P2 |

## 七、知识库建设

### 7.1 知识库是核心资产

知识库不仅存行业标准和规范，更重要的是存每一个项目的资料：
- 地质报告
- 可研报告
- 尽调材料
- 评估报告

**投喂的项目资料越多，AI 就越了解投资组合，回答越精准。**

### 7.2 知识库分类

| 知识库 | 内容 | 来源 |
|--------|------|------|
| 矿业专业知识 | 选矿手册、评估标准、行业规范 | 公众号、杂志、标准文档 |
| 项目档案 | 每个矿山的地质报告、可研等 | 汪总提供 |
| 聊天记录 | 重要对话的归档 | 自动/手动归档 |

### 7.3 初期需要的输入

1. **矿业知识来源**：推荐有价值的公众号、杂志
2. **两个示范项目**：完整的项目资料用于测试
3. **品种列表**：所有希望智能体知道的矿产（氧化铝、铜...）
