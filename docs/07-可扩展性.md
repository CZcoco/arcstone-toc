# 07 - 可扩展性

## 一、概述

Deep Agents 的可扩展性设计确保系统能够：
- 持续添加新的专业技能
- 接入更多外部服务
- 支持未来业务扩展（供应链、硬科技投资）

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    Deep Agents 可扩展性体系                              │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1️⃣ Skills 系统      → 专业技能扩展（估值方法、评估框架）                 │
│  2️⃣ Middleware 系统  → 能力增强（风险预警、价格监控）                    │
│  3️⃣ MCP 协议        → 外部服务接入（数据库、地图、API）                  │
│  4️⃣ SubAgent 系统   → 业务拓展（供应链、硬科技智能体）                   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 二、Skills 系统

### 2.1 什么是 Skills

Skills 是包含专业知识和指令的文件夹，Agent 可以按需动态加载。

**与传统工具的区别**：
- **工具**：底层功能（搜索、计算）
- **技能**：高级指令（如何完成特定任务）

### 2.2 目录结构

```
~/.deepagents/arcstone/skills/      # 全局技能
├── mining-valuation/               # 矿业估值技能
│   ├── SKILL.md
│   └── templates/
│       └── dcf_model.xlsx
├── geological-assessment/          # 地质评估技能

```

### 2.3 SKILL.md 格式

```markdown
---
name: 矿业项目估值
description: 当需要对矿山项目进行 DCF 估值、IRR 计算或可比交易分析时使用此技能
---

# 矿业项目估值指南

## 估值方法选择

根据项目阶段选择合适方法：
- 早期探矿权 → 资源量法（$/吨，对标可比交易）
- 详查阶段 → 可比交易法 + DCF 初步测算
- 可研阶段 → DCF 为主（NPV、IRR、投资回收期）
- 在产矿山 → 收益法（中国矿业权评估准则）

## DCF 模型参数

### 必要输入
- 资源量/储量（吨）
- 品位（%）
- 年产能规划
- CAPEX（采矿+选矿+基建）
- OPEX（采矿+选矿+管理+运输）
- 产品价格假设
- 折现率（通常 8-12%）

### 输出指标
- NPV（净现值）
- IRR（内部收益率）
- 投资回收期
- 盈亏平衡价格

## 敏感性分析

必须进行以下变量的敏感性测试：
- 价格 ±20%
- 品位 ±10%
- 成本 ±15%
- 产能 ±10%

## 模板位置
估值模板见 templates/dcf_model.xlsx
```

### 3.2 自定义中间件

```python
from deepagents.middleware import BaseMiddleware
from langchain.tools import tool

class MiningAlertMiddleware(BaseMiddleware):
    """矿业风险预警中间件"""

    # 扩展状态
    state_schema = {
        "risk_alerts": list,
        "price_thresholds": dict
    }

    # 提供工具
    def get_tools(self):
        return [self.check_price_alert, self.add_risk_alert]

    @tool
    def check_price_alert(self, commodity: str) -> str:
        """检查大宗商品价格是否触及预警阈值"""
        current_price = get_commodity_price(commodity)
        threshold = self.state["price_thresholds"].get(commodity)

        if threshold and current_price < threshold["min"]:
            alert = f"⚠️ {commodity} 价格 {current_price} 已跌破预警线"
            self.state["risk_alerts"].append(alert)
            return alert

        return f"{commodity} 价格正常: {current_price}"

    @tool
    def add_risk_alert(self, project: str, risk: str, severity: str) -> str:
        """添加项目风险提醒"""
        alert = {"project": project, "risk": risk, "severity": severity}
        self.state["risk_alerts"].append(alert)
        return f"已记录风险: {project} - {risk}"

    # 修改系统提示词
    def get_system_prompt_addition(self) -> str:
        return """
## 风险预警指令
在评估过程中，如果发现以下风险请使用 add_risk_alert 记录：
- 品位低于经济开采边界
- 环评/采矿证即将到期
- 价格跌破盈亏平衡点
"""

# 使用
agent = create_deep_agent(
    middleware=[
        MiningAlertMiddleware(),
    ]
)
```

## 四、MCP 协议集成

### 4.1 什么是 MCP

MCP (Model Context Protocol) 是 Anthropic 推出的开放标准，用于连接 AI 与外部工具。

**2025 年行业标准**：OpenAI、Google、Microsoft 都已支持。

### 4.2 LangChain MCP Adapters

```python
from langchain_mcp_adapters import MultiServerMCPClient

async with MultiServerMCPClient(
    {
        # PostgreSQL 数据库
        "postgres": {
            "command": "uvx",
            "args": ["mcp-server-postgres", "--db-url", DB_URL]
        },
        # 高德地图
        "amap": {
            "command": "uvx",
            "args": ["mcp-server-amap", "--api-key", AMAP_KEY]
        },
        # 自定义矿业数据服务
        "mining-data": {
            "command": "python",
            "args": ["mining_mcp_server.py"]
        }
    }
) as client:
    tools = client.get_tools()
    agent = create_deep_agent(tools=tools)
```

### 4.3 可接入的 MCP 服务

| 服务类型 | 示例 | 用途 |
|---------|------|------|
| 数据库 | PostgreSQL、MongoDB | 项目数据存储 |
| 地图服务 | 高德、Google Maps | 距离计算 |
| 文件存储 | Google Drive、S3 | 文档管理 |
| 通讯 | Slack、Email | 通知推送 |
| 自定义 API | 矿价 API | 数据获取 |

## 六、扩展性评估

### 6.1 能力矩阵

| 扩展维度 | 能力 | 成熟度 | 适用场景 |
|---------|------|--------|---------|
| **Skills** | 专业知识注入 | ⭐⭐⭐⭐⭐ | 评估框架、估值方法 |
| **Middleware** | 能力增强 | ⭐⭐⭐⭐⭐ | 风险预警、监控 |
| **MCP** | 外部服务 | ⭐⭐⭐⭐⭐ | 数据库、地图、API |
| **SubAgent** | 业务拓展 | ⭐⭐⭐⭐⭐ | 多业务协作 |
| **自定义工具** | 封装逻辑 | ⭐⭐⭐⭐⭐ | 任意功能 |

### 6.2 框架优势

- **100% 开源（MIT）**：不会被锁定
- **基于 LangGraph**：生态成熟，持续更新
- **LangChain 官方维护**：不会烂尾
- **行业标准（MCP）**：兼容性好

### 6.3 扩展路线图

| 阶段 | 扩展内容 | 时间 |
