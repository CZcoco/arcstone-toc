# 05 - 架构设计：记忆层

## 一、概述

记忆层是让 AI "真正懂用户" 的关键。用户最大的痛点之一就是：**AI 不记得之前讨论过的项目**。

```
┌─────────────────────────────────────────────────────────────┐
│                        记忆体系                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  短期记忆（会话内）     ✅ LangGraph 自动处理                 │
│  ─────────────────────────────────────────────────────────  │
│                                                             │
│  跨对话记忆（核心需求） ✅ Deep Agents 原生支持               │
│  ├── 1. 项目档案记忆   "我们讨论过甘肃镍矿，结论是..."        │
│  ├── 2. 用户偏好记忆   "用户关注 IRR > 15% 的项目"           │
│  ├── 3. 历史决策记忆   "上个月否决了那个铜矿因为环评风险"      │
│  └── 4. 自我改进记忆   "用户纠正过选矿工艺的术语"             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 二、Deep Agents 记忆架构

### 2.1 双层存储设计

Deep Agents 使用 `CompositeBackend` 实现混合存储：

```
┌─────────────────────────────────────────────────────────────┐
│              短期记忆 (StateBackend)                         │
│                                                             │
│   /notes.txt          ← 当前会话笔记                         │
│   /workspace/         ← 临时工作文件                         │
│   /todos.json         ← 当前任务列表                         │
│                                                             │
│   💨 会话结束即消失                                          │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│              长期记忆 (StoreBackend)                         │
│                                                             │
│   /memories/user_profile.md    ← 用户偏好                    │
│   /memories/projects/甘肃镍矿.md ← 项目档案                   │
│   /memories/decisions/         ← 历史决策                    │
│   /memories/instructions.md    ← 自我改进指令                 │
│                                                             │
│   💾 跨会话持久保存                                          │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 路径路由规则

- `/memories/*` 路径 → 持久化存储（StoreBackend）
- 其他路径 → 临时存储（StateBackend）

## 三、配置实现

### 3.1 基础配置

```python
from deepagents import create_deep_agent
from deepagents.backends import CompositeBackend, StateBackend, StoreBackend
from langgraph.store.memory import InMemoryStore  # 开发用
from langgraph.checkpoint.memory import MemorySaver

# 1. 配置检查点（用于中断恢复）
checkpointer = MemorySaver()

# 2. 配置存储（开发环境）
store = InMemoryStore()

# 3. 配置混合存储后端
def make_backend(runtime):
    return CompositeBackend(
        default=StateBackend(runtime),           # 默认：临时存储
        routes={"/memories/": StoreBackend(runtime)}  # /memories/：持久化
    )

# 4. 创建带长期记忆的 Agent
agent = create_deep_agent(
    model=llm,
    store=store,
    backend=make_backend,
    checkpointer=checkpointer,
    tools=tools,
    system_prompt=MINING_SYSTEM_PROMPT
)
```

### 3.2 生产环境配置（SQLite）

对于 Electron 打包的桌面应用，使用 SQLite 本地存储：

```python
from langgraph.store.sqlite import SqliteStore

# 本地 SQLite 持久化
store = SqliteStore.from_path("./data/memories.db")
store.setup()  # 初始化数据库表

agent = create_deep_agent(
    store=store,
    backend=make_backend,
    checkpointer=MemorySaver(),
    # ...
)
```

优势：
- 所有记忆存在本地 `memories.db` 文件
- 用户数据完全私有
- 无需云端服务

### 3.3 多用户环境配置（PostgreSQL）

如果需要支持多用户：

```python
from langgraph.store.postgres import PostgresStore

store = PostgresStore.from_conn_string(os.environ["DATABASE_URL"])
store.setup()
```

## 四、记忆目录结构

### 4.1 推荐的 /memories/ 结构

```
/memories/
├── user_profile.md          # 用户画像
├── instructions.md          # 自我改进指令
├── projects/                # 项目档案
│   ├── 甘肃金川镍矿.md
│   ├── 贵州磷矿项目.md
│   └── 云南铜矿项目.md
├── decisions/               # 历史决策
│   ├── 2025-01-15_甘肃镍矿_观望.md
│   └── 2025-02-01_贵州磷矿_推进尽调.md
└── market_notes/            # 市场观察（可选）
    ├── 铜价趋势.md
    └── 镍市场分析.md
```

### 4.2 文件格式模板

#### user_profile.md

```markdown
# 用户画像

## 投资偏好
- IRR 门槛：15%+
- 偏好地区：西南、西北
- 关注矿种：铜、镍、磷

## 风险关注点
- 环评风险（高度关注）
- 水源问题
- 政策变化

## 决策风格
- 偏保守，喜欢敏感性分析
- 重视长期价值
- 团队协作决策

## 沟通偏好
- 喜欢表格对比
- 希望看到明确的投资建议
- 不喜欢冗长的引用说明

## 更新记录
- 2025-01-15: 确认 IRR 门槛为 15%
- 2025-02-01: 增加对环评风险的关注
```

#### projects/{项目名}.md

```markdown
# 甘肃金川镍矿项目

## 基本信息
- 位置：甘肃省金昌市
- 矿种：硫化镍矿
- 资源量：50万吨金属镍（332+333）
- 品位：0.8% Ni

## 评估历史

### 2025-01-15 首次评估
**评估人**：AI 助手 + 汪总
**主要发现**：
- IRR：12.3%（基准情景）
- 主要风险：水源问题（年降水量 < 200mm）
- 优势：品位中等偏上，开采条件成熟

**结论**：暂时观望
**理由**：IRR 低于 15% 门槛，水源问题需要解决方案

**待办事项**：
- [ ] 让技术团队评估海水淡化可行性
- [ ] 关注镍价走势，价格回升可重新评估

### 2025-02-01 跟进
**状态**：等待技术团队反馈
**新信息**：镍价较上月上涨 5%

## 关键数据
| 指标 | 数值 | 备注 |
|------|------|------|
| 资源量 | 50万吨 | 332+333 |
| 平均品位 | 0.8% | 中等偏上 |
| IRR | 12.3% | 基准情景 |
| NPV (10%) | 8,500万元 | - |
| 回收期 | 6.2年 | - |

## 用户关注点
- 对水源问题很担心
- 希望 IRR 能到 15%+
- 考虑过镍价上涨对项目的影响
```

#### decisions/{日期}_{项目}_{决策}.md

```markdown
# 投资决策记录

## 基本信息
- 日期：2025-01-15
- 项目：甘肃金川镍矿
- 决策：暂时观望

## 决策理由
1. IRR 12.3% 低于 15% 的投资门槛
2. 水源问题是重大风险，年降水量不足 200mm
3. 当前镍价处于相对低位

## 当时的关键数据
- 镍价：120,000 元/吨
- 项目 IRR：12.3%
- 项目 NPV：8,500 万元

## 后续计划
- 等待技术团队的水源解决方案
- 持续关注镍价走势
- 如果镍价上涨 20%，重新评估

## 决策参与人
- 汪总
- AI 助手
```

#### instructions.md

```markdown
# 自我改进指令

## 术语纠正
- 不要说"矿石储量"，应该说"资源量"或"储量"，根据勘探程度区分
- "可研"全称是"可行性研究报告"
- 品位单位要注意：金银用 g/t，铜镍等用 %

## 回答方式
- 用户喜欢表格对比，多个项目对比时优先用表格
- 不需要在回答中显示"[知识库1]"这样的引用
- 给出明确的投资建议，不要模棱两可

## 工作习惯
- 重要结论要主动问是否保存到项目档案
- 发现风险要立即提醒，不要等到最后总结
- 计算 IRR 后要主动做敏感性分析

## 错误记录
- 2025-01-20: 计算 IRR 时忘记考虑建设期，下次注意
- 2025-02-01: 铜价数据用了过时的，要用联网搜索获取最新价格
```

## 五、记忆读写流程

### 5.1 对话开始时读取

```python
# Agent 启动时自动执行
def on_conversation_start(agent, user_message):
    # 1. 读取用户画像
    profile = agent.read_file("/memories/user_profile.md")

    # 2. 检测是否涉及已知项目
    project_name = detect_project_name(user_message)
    if project_name:
        project_file = f"/memories/projects/{project_name}.md"
        project_data = agent.read_file(project_file)

    # 3. 读取自我改进指令
    instructions = agent.read_file("/memories/instructions.md")

    # 注入到上下文
    return f"""
## 用户画像
{profile}

## 项目档案
{project_data if project_name else "无相关项目记录"}

## 工作指令
{instructions}
"""
```

### 5.2 对话过程中写入

在 System Prompt 中指导 Agent 何时写入记忆：

```markdown
# 写入记忆指令

## 用户偏好 → /memories/user_profile.md
当用户表达以下内容时，更新用户画像：
- "我一般只看 IRR 15% 以上的" → 记录 IRR 阈值
- "环评风险我很在意" → 记录风险关注点
- "我喜欢表格对比" → 记录沟通偏好

## 项目讨论 → /memories/projects/{项目名}.md
每次讨论某个项目后，更新该项目档案：
- 新的评估结论
- 新发现的风险
- 待办事项
- 讨论要点摘要

## 重要决策 → /memories/decisions/{日期}_{项目}_{决策}.md
当用户做出投资决策时，记录：
- 决策内容（投资/观望/放弃）
- 决策理由
- 当时的关键数据
- 后续计划

## 自我改进 → /memories/instructions.md
当用户纠正错误或表扬时，更新：
- 术语纠正
- 回答方式偏好
- 错误记录
```

### 5.3 跨会话记忆演示

```python
import uuid

# === 第一次对话（1月15日）===
thread_1 = {"configurable": {"thread_id": str(uuid.uuid4())}}

agent.invoke({
    "messages": [{
        "role": "user",
        "content": "帮我评估甘肃金川镍矿，IRR 低于 15% 的我就不考虑了"
    }]
}, config=thread_1)

# Agent 会：
# 1. 检索知识库获取项目数据
# 2. 计算 IRR = 12.3%
# 3. 发现低于用户门槛
# 4. 写入 /memories/user_profile.md（IRR 门槛）
# 5. 写入 /memories/projects/甘肃金川镍矿.md（评估结论）
# 6. 给出"暂时观望"的建议

# === 一个月后，新对话 ===
thread_2 = {"configurable": {"thread_id": str(uuid.uuid4())}}

response = agent.invoke({
    "messages": [{
        "role": "user",
        "content": "之前那个甘肃镍矿，我们的结论是什么？"
    }]
}, config=thread_2)

# Agent 会：
# 1. 读取 /memories/projects/甘肃金川镍矿.md
# 2. 回答："老朋友，关于甘肃金川镍矿项目，1月15日我们的评估结论是..."
```

## 七、存储方案对比

| 环境 | 存储方案 | 优势 | 劣势 |
|------|---------|------|------|
| 开发测试 | `InMemoryStore` | 简单快速 | 重启丢失 |
| 本地生产 | `SqliteStore` | 无需服务，完全私有 | 单机限制 |
| 云端生产 | `PostgresStore` | 多用户，可扩展 | 需要云服务 |

**推荐**：Arcstone 桌面版使用 SQLite，用户数据完全存在本地。
