# 02 - 框架选型：Deep Agents

## 一、为什么选择 Deep Agents

### 1.1 Agent Loop 的演进

```
传统 Agent Loop（ReAct 模式）：
┌─────────────────────────────────────────────────────────────┐
│   用户输入 → 思考 → 行动 → 观察 → 思考 → 行动 → ... → 输出    │
│                     （单层循环，无规划）                       │
└─────────────────────────────────────────────────────────────┘
问题：容易"浅层"执行，无法处理长期复杂任务

Deep Agents 架构：
┌─────────────────────────────────────────────────────────────┐
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐    │
│  │ 规划系统  │  │ 主循环   │  │  子代理   │  │ 文件系统  │    │
│  │ (Todo)   │→ │ (ReAct)  │→ │ (隔离)   │→ │ (记忆)   │    │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘    │
│       ↑_______________|_______________|___________|          │
│                      反馈循环                                 │
└─────────────────────────────────────────────────────────────┘
优势：规划 + 反思 + 记忆 + 委派
```

### 1.2 Deep Agents 的设计来源

Deep Agents 的设计灵感直接来自 **Claude Code**：

> "Applications like Deep Research, Manus, and **Claude Code** have gotten around this limitation by implementing a combination of four things: a planning tool, sub agents, access to a file system, and a detailed prompt."

LangChain 把这套架构开源打包成 `deepagents` 库，让开发者可以轻松复刻 Claude Code 的体验。

### 1.3 核心特性

| 特性 | 说明 |
|------|------|
| **显式规划** | 内置 Todo 工具，自动分解复杂任务 |
| **持久记忆** | 虚拟文件系统，跨会话保存笔记和档案 |
| **子代理委派** | 生成专门子代理处理子任务，保持主上下文干净 |
| **工具集成** | 无缝使用搜索、API、代码执行等工具 |
| **上下文管理** | 自动摘要压缩，避免上下文溢出 |

## 二、Deep Agents vs 其他框架

### 2.1 主流框架对比

| 框架 | 架构模式 | 核心理念 | 适用场景 | 成熟度 |
|------|----------|----------|----------|--------|
| **Deep Agents** | 单 Agent + 子代理 | Claude Code 复刻 | 通用编码/研究 | ⭐⭐⭐ 新 |
| **LangGraph** | 状态图 | 精确控制流 | 复杂工作流 | ⭐⭐⭐⭐⭐ |
| **AutoGen** | 多代理对话 | 异步消息传递 | 多角色协作 | ⭐⭐⭐⭐ |
| **CrewAI** | 角色团队 | 人类组织模拟 | 快速原型 | ⭐⭐⭐⭐ |
| **OpenAI Agents SDK** | 极简工具调用 | 轻量 + 速度 | 快速开发 | ⭐⭐⭐ 新 |

### 2.2 架构哲学差异

```
AutoGen:    Agent A ←对话→ Agent B ←对话→ Agent C  (像微信群聊)
CrewAI:     Manager → Worker1, Worker2, Worker3    (像公司团队)
LangGraph:  Node1 → [条件] → Node2 or Node3 → ...  (像流程图)
Deep Agents: Main Agent → spawn(子Agent) → 返回    (像 Claude Code)
```

### 2.3 为什么 Deep Agents 最适合我们

| 需求 | Deep Agents 优势 |
|------|-----------------|
| 像 Claude Code 的体验 | 架构直接来源于此 |
| 开箱即用 | `create_deep_agent()` 一行代码 |
| 垂直领域定制 | Skills 系统 + 自定义 Prompt |
| 生产就绪 | 基于 LangGraph，稳定可靠 |
| 未来扩展 | Middleware + MCP + 子代理 |

## 三、Deep Agents 架构详解

### 3.1 三层架构

```
┌─────────────────────────────────────────────────────────────┐
│                       用户接口层                             │
│            CLI 终端 / API 调用 / Studio 集成                 │
├─────────────────────────────────────────────────────────────┤
│                      核心 Agent 框架                         │
│  ┌─────────────┬─────────────┬─────────────┬────────────┐  │
│  │ 规划中间件   │ 文件系统    │ 子代理中间件 │ 摘要中间件  │  │
│  │ TodoList   │ Filesystem  │ SubAgent   │ Summary    │  │
│  └─────────────┴─────────────┴─────────────┴────────────┘  │
├─────────────────────────────────────────────────────────────┤
│                       后端实现层                             │
│           存储后端 / LLM 提供商 / 工具执行环境                │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 内置中间件

| 中间件 | 功能 | 提供的工具 |
|--------|------|-----------|
| `TodoListMiddleware` | 任务规划和跟踪 | `write_todos`, `read_todos` |
| `FilesystemMiddleware` | 虚拟文件系统 | `read_file`, `write_file`, `edit_file`, `ls`, `glob`, `grep` |
| `SubAgentMiddleware` | 子代理委派 | `task` |
| `SummarizationMiddleware` | 上下文压缩 | 自动触发 |
| `HumanInTheLoopMiddleware` | 人工审批 | `interrupt` |

### 3.3 存储后端

| 后端 | 用途 | 持久性 |
|------|------|--------|
| `StateBackend` | 默认后端，存在 Agent 状态中 | 会话内 |
| `StoreBackend` | 持久化后端，存在 LangGraph Store 中 | 跨会话 |
| `CompositeBackend` | 混合路由，部分路径持久化 | 可配置 |

## 四、Deep Agents vs Claude Code

### 4.1 架构对比

| 特性 | Claude Code | Deep Agents | 相似度 |
|------|-------------|-------------|--------|
| 主循环 | 单线程 Master Loop ("nO") | 单线程 ReAct Loop | ⭐⭐⭐⭐⭐ |
| 规划系统 | TodoWrite 工具 | write_todos/read_todos | ⭐⭐⭐⭐⭐ |
| 子代理 | Task Agent (I2A) | task 工具 | ⭐⭐⭐⭐⭐ |
| 文件操作 | Read/Write/Edit/Glob/Grep | 同名工具 | ⭐⭐⭐⭐⭐ |
| 上下文压缩 | compact 命令 | SummarizationMiddleware | ⭐⭐⭐⭐ |
| Skills | SKILL.md 动态加载 | skills 目录支持 | ⭐⭐⭐⭐⭐ |

### 4.2 Claude Code 的设计哲学

```python
# Claude Code 主循环伪代码
while model_response.has_tool_calls():
    results = execute_tools(model_response.tool_calls)
    message_history.append(results)

    # 关键：TODO 提醒注入，防止迷失
    if len(message_history) % N == 0:
        inject_todo_reminder(message_history)

    model_response = claude.chat(message_history)
```

**设计原则**：
- 单线程 + 扁平消息历史（避免多代理混乱）
- TODO 系统作为"外部工作记忆"
- 子代理有深度限制（防止无限递归）
- 透明可调试 > 花哨架构

## 五、技术特点

### 5.1 开源与标准

- **100% 开源**：MIT 许可证，完全免费
- **Provider 无关**：支持 Claude、OpenAI、Google、DeepSeek 等
- **MCP 支持**：通过 langchain-mcp-adapters 接入标准化工具

### 5.2 生产就绪

- 基于 LangGraph 运行时
- 支持流式输出
- 内置持久化和检查点
- 支持 LangSmith 监控

### 5.3 性能表现

在基准测试中，LangGraph 在延迟和 Token 使用方面表现最优：

| 框架 | 延迟 | Token 使用 |
|------|------|-----------|
| LangGraph | 最低 | 最少 |
| CrewAI | 中等 | 中等 |
| AutoGen | 中等 | 中等 |
| LangChain (旧版) | 最高 | 最多 |

## 六、安装与快速开始

### 6.1 安装

```bash
pip install deepagents
# 或
uv add deepagents
```

### 6.2 基础使用

```python
from deepagents import create_deep_agent

# 创建默认 Agent（包含所有内置能力）
agent = create_deep_agent()

# 调用
result = agent.invoke({
    "messages": [{"role": "user", "content": "分析这个代码库的架构"}]
})
```

### 6.3 CLI 工具

```bash
pip install deepagents-cli

# 启动终端 Agent
deepagents
```
