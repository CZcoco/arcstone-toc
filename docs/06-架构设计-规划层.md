# 06 - 架构设计：规划层

## 一、概述

规划层让智能体能够：
- **任务分解**：把复杂问题拆解为可执行步骤
- **进度追踪**：知道做到哪了，还剩什么
- **主动询问**：信息不足时停下来问用户
- **任务委派**：把子任务交给专门的子代理

这是 Deep Agents 区别于普通 Chatbot 的核心能力。

### 2.1 Todo 工具

Deep Agents 内置 `write_todos` 和 `read_todos` 工具：

```python

    {"id": "4", "content": "计算经济性", "status": "pending"},
    {"id": "5", "content": "评估合规风险", "status": "pending"},
    {"id": "6", "content": "输出投资建议", "status": "pending"},
```

### 2.2 规划流程示意

```
用户: "帮我全面评估贵州磷矿项目"
            ↓
┌─────────────────────────────────────────────────────────────┐
│  1. write_todos - 任务分解                                   │
│     □ 读取项目基础资料                                       │
│     □ 评估地质资源（品位、储量）                              │
│     □ 分析开采条件                                          │
│     □ 计算经济性（IRR、NPV）                                 │
│     □ 评估合规风险                                          │
│     □ 输出综合投资建议                                       │
└─────────────────────────────────────────────────────────────┘
            ↓
┌─────────────────────────────────────────────────────────────┐
│  2. 执行第一个任务                                           │
│     → 调用 bailian_rag 检索项目资料                          │
│     → 更新 todos：第一项 ✓                                   │
└─────────────────────────────────────────────────────────────┘
            ↓
┌─────────────────────────────────────────────────────────────┐
│  3. 执行第二个任务                                           │
│     → 分析品位数据                                          │
│     → 发现数据缺失 → interrupt() 询问用户                    │
│     → 用户提供数据 → 继续分析                                │
│     → 更新 todos：第二项 ✓                                   │
└─────────────────────────────────────────────────────────────┘
            ↓
        ... 继续执行 ...
            ↓
┌─────────────────────────────────────────────────────────────┐
│  N. 所有任务完成                                             │
│     → 输出综合评估报告                                       │
│     → 询问是否保存到项目档案                                  │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 动态调整计划

Agent 可以在执行过程中调整计划：

```python
# 原计划
todos = [
    {"id": "1", "content": "读取资料", "status": "completed"},
    {"id": "2", "content": "评估地质", "status": "in_progress"},
    {"id": "3", "content": "计算IRR", "status": "pending"},
]

# 发现新需求，动态添加任务
todos.append({"id": "4", "content": "搜索最新铜价用于计算", "status": "pending"})
write_todos(todos)
```

## 三、Human-in-the-Loop

### 3.1 三种交互模式

| 模式 | 场景 | 实现方式 |
|------|------|---------|
| **询问模式** | 信息不足时主动问 | `interrupt()` 函数 |
| **审批模式** | 敏感操作前确认 | `HumanInTheLoopMiddleware` |
| **编辑模式** | 让用户修改输出 | `Command(resume=edited)` |

### 3.2 询问模式实现

```python
from langgraph.types import interrupt

@tool
def ask_user(question: str, options: list = None) -> str:
    """
    向用户询问信息或确认。
    """
    response = interrupt({
        "type": "question",
        "question": question,
        "options": options or []
    })
    return response
```

#### 使用场景

```python
# 场景1：数据缺失
ask_user("报告中没有找到 P2O5 品位数据，请问您有这个信息吗？")

# 场景2：需要澄清
ask_user(
    "您说的'那个铜矿'是指哪个项目？",
    options=["云南铜矿项目", "西藏铜矿项目", "其他"]
)

# 场景3：决策确认
ask_user(
    "这个项目处于详查阶段，建议用可比交易法+DCF初步测算，您同意吗？",
    options=["同意", "只用DCF", "只用可比交易法"]
)

# 场景4：风险提示
ask_user("我发现这个项目的环评还未通过，这可能影响整体评估，要继续吗？")
```

### 3.3 审批模式实现

```python
from deepagents.middleware import HumanInTheLoopMiddleware

agent = create_deep_agent(
```

### 3.4 前端集成

```javascript
// Arcstone 前端处理中断
async function handleAgentResponse(response) {
    if (response.__interrupt__) {
        const interrupt = response.__interrupt__;

        if (interrupt.type === 'question') {
            // 显示问题对话框
            const answer = await showQuestionDialog({
                question: interrupt.question,
                options: interrupt.options
            });
            return await resumeAgent(answer);
        }

        if (interrupt.type === 'approval') {
            // 显示审批对话框
            const decision = await showApprovalDialog({
                action: interrupt.action,
                details: interrupt.details
            });
            // decision: "approve" | "reject" | "edit"
            return await resumeAgent(decision);
        }
    }

    // 正常响应
    displayMessage(response);
}

// 恢复 Agent 执行
async function resumeAgent(userInput) {
    const response = await fetch('/api/agent/resume', {
        method: 'POST',
        body: JSON.stringify({
            thread_id: currentThreadId,
            resume: userInput
        })
    });
    return response.json();
}
```

## 四、子代理系统

### 4.1 概述

子代理用于：
- **上下文隔离**：避免主代理上下文被污染
- **专业分工**：不同任务用不同专家
- **并行处理**：多个子代理可以同时工作

### 4.2 内置 task 工具

```python
# Agent 可以使用 task 工具委派子任务
task(
    name="researcher",
    prompt="深入研究镍市场的供需情况和价格趋势预测"
)
```

### 4.3 自定义子代理

```python
from deepagents import create_deep_agent

subagents = [
    {
        "name": "researcher",
        "description": "深度研究某个专题，如矿种市场、地区政策",
        "prompt": """你是矿业研究专家，擅长深度调研。
任务：根据用户需求，进行深入研究并输出报告。
要求：引用信息来源，区分事实和观点。""",
        "tools": [web_search, bailian_rag],
        "model": "deepseek-chat"  # 子代理可以用更便宜的模型
    },
    {
        "name": "calculator",
        "description": "执行复杂财务计算和敏感性分析",
        "prompt": """你是财务分析专家。
任务：根据提供的参数进行财务计算。
要求：计算过程清晰，结果准确。""",
        "tools": [calculate_irr, sensitivity_analysis],
        "model": "deepseek-chat"
    },
    {
        "name": "comparator",
        "description": "多项目横向对比分析",
        "prompt": """你擅长对比分析多个矿业项目。
任务：从多个维度对比项目，输出对比表格。
要求：突出各项目优劣势，给出排序建议。""",
        "tools": [bailian_rag, compare_projects]
    }
]

agent = create_deep_agent(
    model="claude-sonnet-4-20250514",  # 主代理用强模型
    subagents=subagents,
    # ...
)
```

### 4.4 子代理使用示例

```
用户: 帮我研究一下全球镍市场的供需情况

Agent: 好的，我会委托研究子代理来深入调研这个专题。

       [启动 researcher 子代理]
       子代理正在搜索相关信息...
       子代理正在整理分析报告...
       [子代理完成]

       根据研究子代理的报告，全球镍市场情况如下：

       ## 供给端
       - 印尼产量持续增长，占全球 50%+
       - 菲律宾出口政策收紧
       - 俄罗斯供应受地缘政治影响

       ## 需求端
       - 新能源汽车带动三元锂电池需求
       - 不锈钢需求稳定
       - 储能市场快速增长

       ## 价格预测
       - 短期（3-6个月）：震荡偏弱
       - 中期（1-2年）：供需趋紧，价格有上涨空间

       需要我基于这些信息重新评估甘肃镍矿项目吗？
```

### 4.5 子代理限制

- 子代理**不能**再生成子代理（防止无限递归）
- 子代理有独立的上下文，不会污染主代理
- 子代理完成后，结果返回给主代理继续处理

## 五、规划层配置

### 5.1 完整配置示例

```python
from deepagents import create_deep_agent
from deepagents.middleware import HumanInTheLoopMiddleware

agent = create_deep_agent(
    model=llm,

    # 工具
    tools=[
        bailian_rag,
        web_search,
        calculate_irr,
        sensitivity_analysis,
        ask_user,
    ],

    # 子代理
    subagents=[
        {"name": "researcher", ...},
        {"name": "calculator", ...},
    ],

    # 中间件
    middleware=[
        HumanInTheLoopMiddleware(
            interrupt_on={
                "write_file": True,
                "confirm_action": True,
            }
        )
    ],

    # 存储
    store=store,
    backend=make_backend,
    checkpointer=checkpointer,

    # 提示词
    system_prompt=MINING_SYSTEM_PROMPT,
)
```

### 5.2 Prompt 中的规划指令

```markdown
# 任务规划指令

## 何时创建任务列表
当用户的请求涉及多个步骤时，使用 write_todos 创建任务列表：
- "帮我评估这个项目" → 需要多步骤评估
- "对比这几个矿" → 需要检索、对比、总结
- "计算 IRR 并做敏感性分析" → 需要计算、分析

## 单步任务不需要创建列表
- "铜价现在多少" → 直接搜索回答
- "这个报告里品位是多少" → 直接检索回答

## 任务执行原则
1. 按顺序执行，完成一个标记一个
2. 遇到问题时暂停询问，不要跳过
3. 可以根据新信息调整计划
4. 每完成一个阶段性成果，询问用户是否继续
```
